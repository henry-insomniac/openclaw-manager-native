# Imported Skills

These skills were copied from `~/.agents/skills` into this repo so UI and UX work can run against a local project context.

## Primary

- `frontend-design`
  Use for layout, hierarchy, color, spacing, motion, and responsiveness.

- `clarify`
  Use for simplifying labels, explanations, button text, diagnostics copy, and empty states.

- `polish`
  Use for final spacing, alignment, consistency, interaction state, and visual cleanup.

## Supporting

- `quieter`
  Use when the interface feels too loud, too dense, or too card-heavy.

- `adapt`
  Use when a component needs to work better across narrow and wide window sizes.

- `animate`
  Use sparingly for purposeful motion after hierarchy and readability are already solid.

- `find-skills`
  Use when you want to search for more project-relevant skills.

## Expected Workflow

1. Read `frontend-design` first for UI work.
2. Use `clarify` to tighten copy and button labels.
3. Use `quieter` if the result still feels visually noisy.
4. Use `polish` before shipping.
