# openclaw-manager-native

Before making substantial changes in this repo, read every file under the project `.claude/` directory.

Start here:

1. `.claude/README.md`
2. `.claude/UI.md`
3. `.claude/skills/INDEX.md`

Then open the full skill docs that match the task.

Rules for this repo:

- This product exists to lower the technical cost of local OpenClaw troubleshooting and maintenance.
- Default audience is non-technical or semi-technical users. Write and design for people who do not want to live in Terminal.
- UI copy should be short, direct, Chinese-first, and action-oriented. Avoid marketing tone and repeated explanations.
- Prioritize information hierarchy. Show current judgment first, impact second, and the safest next action third.
- Avoid visual noise: repeated cards, repeated headings, and repeated facts should be collapsed or flattened.
- Preserve the native app's dark utility aesthetic, but keep it calmer and more legible than a generic dashboard.
- For UI work, use the imported design skills in `.claude/skills/` instead of inventing a new style each time.
