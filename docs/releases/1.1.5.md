# OpenClaw Manager Native 1.1.5

发布日期：2026-03-17

## 这版改了什么

- 这是 `1.1.4` 之后的运行时切换收口版，重点把默认位从“只复制当前账号 auth”改成“同步整个 managed Codex auth pool”，并把 runtime 真实正在跑的 auth profile 直接暴露给原生 UI。
- 这版同时补了运行时限流退避：如果 gateway 日志里刚出现 API rate limit，manager 会把同账号的 profile 暂时标成 `cooldown`，避免推荐结果马上切回刚被限流的账号。

## 修复点

### 1. 默认位开始维护多账号 Codex auth pool

- `syncProfileToDefault` 不再把默认位 runtime auth 收窄成单账号副本，而是汇总所有已发现的 managed Codex profile
- 默认位 `auth-profiles.json` 和 `openclaw.json.auth.profiles/order` 现在会生成一组 `openai-codex:<profile>` runtime profile
- 当前 active 账号会排在 `auth.order.openai-codex` 首位，同时继续保留默认位已有的非 Codex auth 配置和 usageStats

### 2. 概览和菜单栏直接显示“运行 auth”

- daemon 现在会从默认位 auth store 的 `lastGood + usageStats.lastUsed` 推导最近成功使用的 runtime auth profile
- 推导结果会尽量映射回 managed profile 名称、邮箱和账号 ID，并通过 `runtime.switching.currentAuthSelection` 暴露给 Swift
- overview 概览、hero meta 和菜单栏 tooltip / 状态区现在都能直接看到当前 runtime 正在跑哪个 auth

### 3. 最近命中过限流的账号会短暂退避

- daemon 会从 `~/.openclaw/logs/gateway.err.log` 读取最近 runtime 失败信号，优先识别 API rate limit
- 一旦发现最近切换后的活跃账号刚被限流，会把同账号镜像到默认位的 profile 一并标成 `cooldown`
- cooldown 会写入 runtime telemetry，在冷却窗口内保持推荐结果避开该账号，减少“刚切走又切回去”的来回抖动

## API / UI

- `GET /api/openclaw/manager` 新增 `runtime.switching.currentAuthSelection`
- `profiles[].status` 现在可能因为最近 runtime rate limit 临时变成 `cooldown`
- 原生 UI 的 overview 和菜单栏已同步消费上述字段
