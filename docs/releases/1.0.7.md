# OpenClaw Manager Native 1.0.7

发布日期：2026-03-12

## 这版改了什么

- 原生 app 新增独立机器监控页，开始把“有 OpenClaw”与“没有 OpenClaw”两种场景统一到一个桌面工具里。
- 监控页加入动态趋势、占用前 10 进程和跳转 Activity Monitor，顶部工具栏也能直接看到机器速览。
- 风险文案和判定降噪，不再把正常的 swap 残留、代理/VPN 等轻微信号直接渲染成高风险。
- 启动链和缓存策略继续收口，普通读取不再因为缓存刚过期就卡住 2 到 4 秒。

## 机器监控

### 1. 新增独立监控摘要接口

- daemon 新增 `GET /api/machine/summary`
- 指标范围：
  - OpenClaw 可用性
  - CPU
  - 内存
  - swap
  - 磁盘剩余
  - 网络吞吐
  - manager / watchdog 进程状态

### 2. 监控页加入动态趋势

- CPU
- 内存压力
- swap
- 下载
- 上传

监控页使用独立 `monitorOnly` 刷新范围，避免把高频刷新拖回完整诊断链路。

### 3. 新增占用前 10 进程

- 按 CPU 排序
- 展示进程名、PID、CPU、RSS、运行时长和命令行
- 可直接打开 macOS 活动监视器继续查看

### 4. 顶部工具栏加入机器速览

- 所有页面都能看到当前机器摘要
- 展示 CPU / 内存压力 / swap / OpenClaw / Manager / 网络上下行
- 同时提供 CPU / 内存 / 网络小趋势图

## 诊断和文案降噪

### 1. 机器压力改成更稳的“内存状态”

- 以内存压力为主
- swap 只作为次级信号
- `31%` 压力 + `93%` swap 不再直接渲染成高压红色

### 2. 环境判定降级

- 新增 `notice / watch / high` 三档
- 单一环境信号只给提示
- 多信号才进入观察
- 只有更硬的网络出口异常才标高风险

### 3. 诊断文案更中性

- `机器压力` -> `内存状态`
- `环境风险` -> `环境因素`
- 避免把历史残留和轻微信号写成当前故障

## 性能修复

### 1. 启动改成 staged loading

- 先 `managerOnly`
- 再 `monitorOnly`
- support 摘要延后后台预热

效果：

- 首屏不再被完整诊断链路阻塞
- manager 和主界面更早可用

### 2. support / machine 非 fresh 路径改成 stale-while-revalidate

- 有缓存时先返回缓存
- TTL 过期后后台续刷
- 不再让普通读取同步等待完整采集

修复前后实测：

- `/api/support/summary` 首次 TTL 过期普通读取：`3810.07 ms` -> `1.06 ms`
- `/api/machine/summary` 首次 TTL 过期普通读取：`2194.69 ms` -> `1.93 ms`

### 3. 默认刷新范围更保守

- 顶部“刷新”按当前页面 scope 执行
- 总览 / 设置 / 账号池等普通页面不再默认 full
- 创建 profile、检查 profile、登录流程完成后也不再无条件完整刷新
