# OpenClaw Manager Native 1.1.3

发布日期：2026-03-13

## 这版改了什么

- 这是 `1.1.2` 之后的 hotfix，重点收 skills 市场“详情经常超时”和“结果被硬截断”两类高频使用问题。
- 这版没有继续扩功能，主要是把详情链路缩短、把失败回退补齐，并把市场结果改成可渐进展开。

## 修复点

### 1. skills 市场详情超时 hotfix

- daemon 的 `GET /api/openclaw/skills/market/{slug}` 不再为了详情再同步拉一次默认市场榜单
- 详情链路现在直接请求 ClawHub 详情，减少一次远程依赖和一段额外等待
- 如果 ClawHub 详情临时失败，会先回退到已有详情缓存；没有详情缓存时，再回退到当前市场缓存里的 item 摘要

### 2. 原生壳补上同 slug 防重入

- Swift 端 `loadSkillMarketDetail` 现在会拦住同一个 skill 的重复详情请求
- 本地 API 等待时间从默认 `8s` 提高到 `20s`
- 连续点选或快速切回同一 skill 时，不再更容易把自己点进假性超时

### 3. 市场结果改成渐进展开

- 默认浏览场景仍保持轻量首屏渲染，避免输入和切换时整块列表重排
- 但结果列表不再只是硬截断到前 `96 / 180` 项
- 现在支持“再显示 N 项 / 显示全部”，并会把当前详情项自动拉回可见范围
