# OpenClaw Manager Native 1.1.4

发布日期：2026-03-16

## 这版改了什么

- 这是 `1.1.3` 之后的修复版，重点处理两个高频误导点：全屏关闭黑屏，以及账号池把认证令牌时效说成“剩余时效”。
- 这版同时把上一轮已完成但未正式发版的 profile 受控编辑补进正式交付，让 inactive profile 也能走同一套 preview / apply / validate / rollback 链路。

## 修复点

### 1. 全屏关闭不再留下黑屏壳

- 原生壳现在会区分普通窗口关闭和 full screen 关闭
- 普通窗口继续隐藏回菜单栏
- full screen 状态点击关闭时，会先退出全屏，再隐藏窗口，避免留下一整个黑色空间

### 2. 账号池把认证时效和额度窗口拆开

- 账号池原来展示的“剩余时效”实际是 OAuth 访问令牌剩余时间，不是账号套餐时效
- 现在改成明确的 `认证剩余` / `认证到期`
- 详情区补了一条短说明，明确告诉用户：真正可用额度看 `5 小时剩余` 和 `本周剩余`
- API 文档也同步补充了 `tokenExpiresAt` / `tokenExpiresInMs` 与 `quota.fiveHour` / `quota.week` 的语义区别

### 3. profile 受控编辑补到 inactive 账号

- daemon 的 profile config preview / apply 不再只限 active profile
- 对 inactive profile 写入时，只改目标账号自己的 `openclaw.json`
- 写入后仍会走 validate 和回滚保护，但不会立刻切换当前 active profile
- Swift 端配置卡片也补上了“切换后生效”的明确文案
