# OpenClaw Manager Native 1.1.0

发布日期：2026-03-13

## 这版改了什么

- 原生 app 正式进入 OpenClaw 本地配置中心阶段，重点能力已经从“看状态、做诊断”扩展到“直接可视化管理 skills”。
- 新增技能工作台，覆盖 skills 市场浏览、本地库存、安装、卸载、启用、停用和配置编辑，日常管理不再依赖手改 `openclaw.json`。
- Profiles 页新增 active profile 安全编辑，支持白名单字段预览、应用、冲突控制、显式校验和失败回滚。

## 重点一：可视化管理 Skills

### 1. 新增独立技能工作台

- 顶级“技能”入口改成独立工作面
- 支持“浏览市场 / 本地库存”切换
- 市场和库存都支持搜索

### 2. 技能市场可直接安装

- daemon 新增 skills 市场、详情、库存、安装、卸载接口
- 市场目录来自 `awesome-openclaw-skills`
- 详情和下载来自 ClawHub
- 安装目录统一落到 Manager 托管目录

### 3. 本地库存可以直接启用、停用和卸载

- manager-owned skill 可直接卸载
- 已安装 skill 可直接启用 / 停用
- bundled skill 支持“允许并启用”

### 4. Skills 配置可直接编辑

- 设置页可查看并编辑 `skills.load.extraDirs`
- 支持新增挂载目录、打开目录、移除目录
- 支持保存 `watch` / `watchDebounceMs`
- 支持保存 `install.preferBrew` / `install.nodeManager`

### 5. 本地库存结构更清楚

- 区分 `市场安装 / 本地个人 / bundled`
- personal skills 不再混成 external
- manager 托管目录会明确标注，避免误删安装根目录

## 重点二：Profile 配置开始支持安全编辑

### 1. 新增 preview / apply 闭环

- `POST /api/openclaw/profiles/{profileName}/config/preview`
- `POST /api/openclaw/profiles/{profileName}/config/apply`

### 2. 只开放白名单字段

- `primaryProviderId`
- `primaryModelId`
- `authMode`

### 3. 应用过程更保守

- 只允许编辑当前激活中的 profile
- 请求必须带 `config/document` 返回的 `baseHash`
- `baseHash` 不匹配时拒绝覆盖
- 写回前先备份原始 `openclaw.json`
- 应用后立即执行 validate
- validate 失败自动回滚
