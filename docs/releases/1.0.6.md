# OpenClaw Manager Native 1.0.6

发布日期：2026-03-12

## 这版改了什么

- 原生 app 继续清掉说明性和宣传性文案，页面只保留状态、动作和必要字段。
- 原生 app 配色从偏蓝改成更暗的石墨绿 / 灰褐色系，减少高饱和高亮边框和无意义强调。
- 官网首页重写成更短的功能说明页，只保留功能、常见问题、下载、更新和 GitHub。
- GitHub 入口保留在官网顶部导航和首屏按钮，下载仍然走 GitHub Releases `latest` 资产。

## 原生 app UI 调整

### 1. 顶部总览 Hero 移除

- 不再在每个页面顶部重复显示大段总览卡片
- 删掉了重复的按钮堆叠、状态 pill 和说明性标题
- 页面直接进入当前功能区

### 2. 诊断页收成“状态 + 动作 + 字段”

移除的说明块：

- `为什么重要`
- `建议先做` 的解释文案
- `技术原文`
- `维护建议`
- `一句话结论`
- `底层说明`

保留的内容：

- 当前判断
- 下一步动作
- 维护动作按钮
- 详细状态折叠区
- 必要的事实字段

### 3. 设置、命令、账号池同步删字

- 设置页去掉副标题和解释句
- 命令页去掉“兼容性 / 常用入口 / 当前使用的路径”这类说明型副标题
- 账号池页把 `当前焦点账号 / 快速切换焦点 / Codex 登录流程 / 新建账号槽位` 等标题压短
- 加载态统一压成简短的 `读取中`

### 4. 文案改成更直接的功能词

- `provider` -> `模型来源`
- `Companion` -> `外部命令行`
- `探测` -> `检查`
- `探测窗口` -> `检查窗口`
- `5小时额度 / 周额度` -> `5 小时剩余 / 本周剩余`

### 5. 暗色调收口

- 主色从偏蓝切成更暗的石墨绿 / 灰褐色
- 降低了高亮描边、对比过强的蓝色按钮和过亮状态块
- 侧栏、卡片、选中态和主按钮统一到更克制的暗色体系

## 官网内容调整

### 删除的区块和文案

- 首屏 GitHub callout
- release strip
- 底部 CTA
- 多余的 eyebrow / kicker / 推荐式标签
- 营销式标题和口号句

### 保留的结构

- 功能
- 常见问题
- 下载
- 更新记录
- FAQ
- GitHub

### 仍然保留的 GitHub 入口

- 顶部导航 `GitHub`
- 首屏按钮 `GitHub`
- 下载区 `GitHub Release`

## 版本和分发

- app 版本升到 `1.0.6`
- 官网展示版本同步到 `1.0.6`
- 下载链接继续使用 GitHub Releases 的 `latest/download`
- 当前 release 页面切到 `v1.0.6`
