# OpenClaw Manager Native 1.1.1

发布日期：2026-03-13

## 这版改了什么

- 这是 `1.1.0` 之后的 hotfix，专门修复 manager-owned skills 在切换 active profile 后可能整体“消失”的回归。
- 问题根因不是技能被删掉，而是默认 `openclaw.json` 在同步 profile 时把原本已经写入的 `skills.load.extraDirs` 覆盖掉了，导致 runtime 不再扫描 Manager 托管目录。
- 这版会在 profile 同步时保留默认配置里的 `skills` 顶层域，并在发现 Manager 托管 skills 仍存在但挂载目录丢失时自动补回挂载。

## 修复点

### 1. profile 同步不再覆盖默认 skills 配置

- `syncProfileToDefault` 不再直接整份复制 profile 配置
- 同步到默认 `openclaw.json` 时会保留原有顶层 `skills` 配置
- 已存在的 `skills.load.extraDirs`、`skills.entries.*` 与相关 watcher / install 设置不会再被 profile 同步链路抹掉

### 2. manager-owned skills 挂载支持自动自愈

- daemon 在构建 skills summary / inventory 前会先检查 Manager 托管目录是否仍有已安装技能
- 如果技能目录和 lock / origin 仍在，但默认 `openclaw.json` 缺少对应 `skills-market` 挂载目录，会自动把托管目录重新写回 `skills.load.extraDirs`
- 自愈后会自动失效旧缓存，避免界面继续显示过期状态
