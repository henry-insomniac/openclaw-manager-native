# OpenClaw Manager Native 1.1.2

发布日期：2026-03-13

## 这版改了什么

- 这是 `1.1.1` 之后的 hotfix，重点收口 skills 市场这一轮剩余的可用性问题。
- 市场默认列表和搜索不再混用 `awesome-openclaw-skills` 与 ClawHub，而是统一切到 ClawHub 官方 API。
- 同时补上了原生 `编辑` 菜单、中文摘要展示，以及市场搜索输入时的列表渲染性能修复。

## 修复点

### 1. skills 市场主源切到 ClawHub

- `GET /api/openclaw/skills/market` 默认列表现在走 ClawHub `/api/v1/skills`
- 市场搜索现在走 ClawHub `/api/v1/search`
- 默认 browse 只抓热门分页，不再把整个市场首页退回到 GitHub markdown 目录
- skill 详情页链接统一指向 `https://clawhub.ai/skills/{slug}`

### 2. 搜索框补齐系统文本命令链

- 原生菜单新增 `编辑`
- 已补齐 `撤销 / 重做 / 剪切 / 复制 / 粘贴 / 全选`
- skills 市场搜索框现在可以正常粘贴文本并直接回车搜索

### 3. 中文摘要与中文搜索回退

- 市场 item / detail 新增 `summaryZh`
- 当前先用本地规则生成中文友好摘要，不依赖外部翻译服务
- 详情页默认优先展示中文摘要，并保留“原文说明”折叠区
- 当 ClawHub 官方搜索结果不足时，会用热门列表的 `summaryZh` 做本地中文回退匹配

### 4. 输入性能热修复

- 市场结果区改成惰性渲染
- 默认 browse 场景只先渲染前 `96` 项，避免输入时整块列表一起重排
- 搜索或分类筛选场景仍可放宽到 `180` 项
- 当前 UI 会明确提示“先渲染前 N 项”，避免看起来像结果被截断
